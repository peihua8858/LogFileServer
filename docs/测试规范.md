# LogFileServer 测试规范

> 本文档为 LogFileServer（Kotlin + Spring Boot）项目的单元测试规范，供团队日常开发参考。

---

## 1. 测试框架与依赖

### 当前已有依赖（build.gradle）

```groovy
testImplementation 'org.springframework.boot:spring-boot-starter-security-test'
testImplementation 'org.springframework.boot:spring-boot-starter-websocket-test'
testImplementation 'org.jetbrains.kotlin:kotlin-test-junit5'
testRuntimeOnly 'org.junit.platform:junit-platform-launcher'
```

> `kotlin-test-junit5` 已包含 JUnit 5 引擎和 `kotlin.test` 断言（`assertEquals`、`assertTrue` 等），可直接使用。

### 建议额外引入：MockK

Kotlin 项目中，Mockito 对 `final class`、`data class`、`suspend` 函数支持不佳。**MockK** 是 Kotlin 生态的首选 Mock 框架，语法原生契合 Kotlin。

在 `build.gradle` 中追加：

```groovy
testImplementation 'io.mockk:mockk:1.13.13'
```

> **注意**：暂不引入 AssertJ、SpringMockK 等其他库。`kotlin.test` 断言 + MockK 已能满足全部单元测试需求。

## 3. 单元测试规范

### 3.1 命名规范

**测试方法**：使用反引号包裹中文或英文行为描述（Kotlin 风格），优先推荐中文：

```kotlin
@Test
fun `应正确解析APK文件的包名和版本号`() { ... }

@Test
fun `当文件为空时应返回错误信息`() { ... }
```

也可使用 `should_` 英文风格（团队统一即可）：

```kotlin
@Test
fun `should return error when file is empty`() { ... }
```

### 3.2 测试结构

使用 `@Nested` 按功能分组，`@DisplayName` 提供可读的中文分组名：

```kotlin
@Nested
@DisplayName("APK 文件解析")
inner class ApkParsing {

    @Test
    fun `应正确提取包名`() { ... }

    @Test
    fun `应正确提取版本号`() { ... }
}
```

### 3.3 AAA 模式

每个测试方法严格遵循 **Arrange-Act-Assert** 三段式，用空行分隔：

```kotlin
@Test
fun `当buildType为空时应默认为Develop`() {
    // Arrange
    val tempApk = createTempApkFile()
    val parameter = Parameter(file = tempApk, buildType = "", desc = "", isOnlyUploadFile = false, isOverWrite = false)

    // Act
    val (appInfo, _) = ApkParser().onParser(parameter)

    // Assert
    assertEquals("Develop", appInfo.buildType)
}
```

### 3.4 MockK 使用规范

#### 基本 Mock

```kotlin
// Mock 接口
val appInfosService = mockk<AppInfosService>()
every { appInfosService.getById(1L) } returns mockAppInfo()

// Mock 普通类（允许 mock final class，MockK 默认支持）
val parser = mockk<ApkParser>()
every { parser.onParser(any()) } returns (AppInfo() to null)
```

#### mockkObject —— mock 单例 / companion object

```kotlin
mockkObject(IParser.Companion)
every { IParser.createParser<AppInfo>("apk", any()) } returns mockk()
```

#### verify —— 验证调用

```kotlin
verify(exactly = 1) { appInfosService.save(any()) }
verifyOrder {
    service.parseFile(any())
    service.saveAppInfo(any())
}
```

#### relaxed mock —— 对未定义的调用返回默认值

```kotlin
val service = mockk<SaveAppFileService>(relaxed = true)
// 调用任何未 stub 的方法不会抛异常，返回 null / 0 / false
```

#### @MockK 注解（推荐用于字段较多时）

```kotlin
@ExtendWith(MockKExtension::class)
class AppControllerTest {

    @MockK
    private lateinit var appInfosService: AppInfosService

    @MockK
    private lateinit var saveAppFileService: SaveAppFileService

    @BeforeEach
    fun setUp() {
        // MockKExtension 已自动初始化 @MockK 字段
    }
}
```

### 3.5 数据驱动测试

使用 JUnit 5 的 `@ParameterizedTest`：

```kotlin
@ParameterizedTest(name = "平台: {0} → 期望: {1}")
@CsvSource(
    "Android, true",
    "iOS,     false",
    "ios,     false",
)
fun `wasAndroid应根据platform返回正确结果`(platform: String, expected: Boolean) {
    val appInfo = AppInfo(platform = platform)
    assertEquals(expected, appInfo.wasAndroid())
}
```

## 4. 测试数据管理

### 4.1 Fixture 工厂方法

在测试类或公共测试辅助文件中提供构建测试数据的工厂方法：

```kotlin
// TestFixtures.kt
object TestFixtures {

    fun mockAppInfo(
        id: Long = 1L,
        bundleId: String = "com.example.app",
        versionName: String = "1.0.0",
        versionCode: Int = 1,
        platform: String = "Android",
        buildType: String = "release",
    ) = AppInfo(
        id = id,
        bundleId = bundleId,
        versionName = versionName,
        versionCode = versionCode,
        platform = platform,
        buildType = buildType,
        name = "TestApp",
    )
}
```

### 4.2 临时测试文件

```kotlin
import kotlin.io.path.createTempFile

fun createTempApkFile(): File {
    // 从 src/test/resources/testdata/ 复制一个小型样本 apk
    val resource = File("src/test/resources/testdata/sample.apk")
    val temp = createTempFile(suffix = ".apk").toFile()
    resource.copyTo(temp, overwrite = true)
    return temp.apply { deleteOnExit() }
}
```

> **规范**：测试资源文件应尽可能小（< 1MB），并在 `deleteOnExit()` 或 `@AfterEach` 中清理。



## 6. 示例代码

### 6.1 Parser 层 —— ApkParserTest

```kotlin
package com.peihua8858.logfileserver.fileparser.impl

import com.peihua8858.logfileserver.fileparser.Parameter
import kotlin.test.*
import java.io.File

class ApkParserTest {

    private lateinit var tempApk: File

    @BeforeEach
    fun setUp() {
        // 从测试资源复制样本 apk
        tempApk = File("src/test/resources/testdata/sample.apk")
            .copyTo(kotlin.io.path.createTempFile(suffix = ".apk").toFile(), overwrite = true)
            .apply { deleteOnExit() }
    }

    @AfterEach
    fun tearDown() {
        tempApk.delete()
    }

    @Nested
    @DisplayName("APK 基础信息解析")
    inner class BasicParsing {

        @Test
        fun `应正确解析包名`() {
            // Arrange
            val parser = ApkParser()
            val parameter = Parameter(file = tempApk, buildType = "release",
                desc = "", isOnlyUploadFile = false, isOverWrite = false)

            // Act
            val (appInfo, _) = parser.onParser(parameter)

            // Assert
            assertNotNull(appInfo.bundleId)
            assertTrue(appInfo.bundleId!!.isNotBlank())
        }

        @Test
        fun `当buildType为空时应默认设为Develop`() {
            val parameter = Parameter(file = tempApk, buildType = "",
                desc = "", isOnlyUploadFile = false, isOverWrite = false)

            val (appInfo, _) = ApkParser().onParser(parameter)

            assertEquals("Develop", appInfo.buildType)
        }

        @Test
        fun `platform应固定为Android`() {
            val parameter = Parameter(file = tempApk, buildType = "debug",
                desc = "", isOnlyUploadFile = false, isOverWrite = false)

            val (appInfo, _) = ApkParser().onParser(parameter)

            assertEquals("Android", appInfo.platform)
        }
    }

    @Nested
    @DisplayName("图标解析")
    inner class IconParsing {

        @Test
        fun `应返回图标数据或null`() {
            val parameter = Parameter(file = tempApk, buildType = "release",
                desc = "", isOnlyUploadFile = false, isOverWrite = false)

            val (_, iconData) = ApkParser().onParser(parameter)

            // iconData 可以为 null（当 apk 无 png 图标时），但不应抛异常
            // 若有图标则应为非空字节数组
            if (iconData != null) {
                assertTrue(iconData.isNotEmpty())
            }
        }
    }
}
```

### 6.2 Service 层 —— SaveAppFileServiceImplTest（使用 MockK）

```kotlin
package com.peihua8858.logfileserver.services.appinfo.impl

import com.peihua8858.logfileserver.entity.appinfo.AppInfo
import com.peihua8858.logfileserver.fileparser.ResultData
import io.mockk.*
import jakarta.servlet.http.HttpServletRequest
import org.junit.jupiter.api.BeforeEach
import org.junit.jupiter.api.Nested
import org.junit.jupiter.api.Test
import org.springframework.mock.web.MockMultipartFile
import kotlin.test.assertEquals
import kotlin.test.assertTrue
import kotlin.test.assertNotNull

class SaveAppFileServiceImplTest {

    private lateinit var service: SaveAppFileServiceImpl
    private val request: HttpServletRequest = mockk(relaxed = true)

    @BeforeEach
    fun setUp() {
        service = spyk(SaveAppFileServiceImpl())
        every { request.scheme } returns "http"
        every { request.serverName } returns "localhost"
        every { request.serverPort } returns 8080
    }

    @Nested
    @DisplayName("jenkinsUploadFile 场景")
    inner class JenkinsUpload {

        @Test
        fun `当文件为空时应返回错误ResultData`() {
            // Arrange
            val emptyFile = MockMultipartFile("file", "app.apk",
                "application/vnd.android.package-archive", ByteArray(0))

            // Act
            val result = service.parserAndSaveFile(
                request, "", "release",
                isOverWrite = false, isOnlyUploadFile = false, file = emptyFile
            )

            // Assert
            assertTrue(result.error.isNotBlank())
        }
    }

    @Nested
    @DisplayName("批量上传场景")
    inner class BatchUpload {

        @Test
        fun `应返回与文件数量相同的ResultData列表`() {
            val files = arrayOf(
                MockMultipartFile("files", "a.apk", "application/octet-stream", "fake".toByteArray()),
                MockMultipartFile("files", "b.ipa", "application/octet-stream", "fake".toByteArray()),
            )

            // 使用 spyk 拦截单个文件处理，避免真实文件操作
            val mockResult = ResultData(name = "a.apk", url = "files/upload/android/a.apk")
            every {
                service.parserAndSaveFile(request, any(), any(), any(), any(), any())
            } returns mockResult

            val result = service.parserAndSaveFiles(
                request, "desc", "release", false, false, files
            )

            assertNotNull(result["files"])
            assertEquals(2, result["files"]!!.size)
        }
    }
}
```

### 6.3 Controller 层 —— AppControllerTest（纯 MockK，不启动 Spring 容器）

```kotlin
package com.peihua8858.logfileserver.controller

import com.peihua8858.logfileserver.entity.appinfo.AppInfo
import com.peihua8858.logfileserver.services.appinfo.AppInfosService
import com.peihua8858.logfileserver.services.appinfo.SaveAppFileService
import io.mockk.*
import org.junit.jupiter.api.BeforeEach
import org.junit.jupiter.api.Nested
import org.junit.jupiter.api.Test
import org.junit.jupiter.api.extension.ExtendWith
import io.mockk.junit5.MockKExtension
import org.springframework.ui.Model
import jakarta.servlet.http.HttpServletRequest
import kotlin.test.assertEquals

@ExtendWith(MockKExtension::class)
class AppControllerTest {

    private lateinit var controller: AppController

    @MockK
    private lateinit var appInfosService: AppInfosService

    @MockK
    private lateinit var saveAppFileService: SaveAppFileService

    @MockK(relaxed = true)
    private lateinit var request: HttpServletRequest

    @MockK(relaxed = true)
    private lateinit var model: Model

    @BeforeEach
    fun setUp() {
        controller = AppController(appInfosService, saveAppFileService)
    }

    @Nested
    @DisplayName("首页列表")
    inner class HomePage {

        @Test
        fun `应返回首页视图`() {
            every { request.serverName } returns "localhost"
            every { appInfosService.queryLastTimeAndMaxVersionByPlatform(any()) } returns mutableListOf()
            every { model.addAttribute(any(), any()) } returns model

            val viewName = controller.homePage(request, model, "", 0)

            assertEquals("/pc/app/app_home", viewName)
            verify { model.addAttribute("result", any()) }
        }

        @Test
        fun `当v=1时应调用queryLastTimeUpdateByPlatform`() {
            every { request.serverName } returns "localhost"
            every { appInfosService.queryLastTimeUpdateByPlatform(any()) } returns mutableListOf()
            every { model.addAttribute(any(), any()) } returns model

            controller.homePage(request, model, "Android", 1)

            verify(exactly = 1) { appInfosService.queryLastTimeUpdateByPlatform("Android") }
            verify(exactly = 0) { appInfosService.queryLastTimeAndMaxVersionByPlatform(any()) }
        }
    }

    @Nested
    @DisplayName("安装引导页")
    inner class InstallGuide {

        @Test
        fun `应返回安装引导视图`() {
            val viewName = controller.installGuide(model)
            assertEquals("/pc/app/install_guide", viewName)
        }
    }
}
```

## 7. 注意事项

| 项目 | 说明 |
|------|------|
| 避免真实文件 IO | 涉及文件操作的测试，优先使用临时文件或 mock，测试结束后清理 |
| 不依赖外部网络 | 单元测试不应调用真实 HTTP 接口；需要时用 `OkHttp MockWebServer` |
| 保持快速 | 单元测试应毫秒级完成；慢测试说明 mock 不充分 |
| 一个测试一个断言主题 | 每个 `@Test` 方法只验证一个行为或一组紧密相关的行为 |
| 清理副作用 | 使用 `@AfterEach` 清理临时文件、重置 mock 状态 |
| `@MockK` vs `mockk()` | 字段多、多方法共用时用注解；局部单次使用用函数 |
